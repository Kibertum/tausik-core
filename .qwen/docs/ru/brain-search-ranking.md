# Ранжирование поиска brain (`brain_search`)

Локальные попадания используют SQLite FTS5 **bm25** (`brain_search.search_local`): чем **меньше** `score`, тем выше релевантность.

Если в MCP передан **`prefer_stack`** (список строк, например `python`, `react`):

1. Локальный путь может вернуть до **`min(limit × 5, 100)`** кандидатов, чтобы сильное совпадение по стеку не отсеялось до пересортировки.
2. После объединения с опциональными результатами Notion список сортируется по **эффективному score**  
   `effective = bm25_score − stack_boost`, где `stack_boost` растёт при пересечении поля `stack` записи с предпочитаемой меткой (точное совпадение или подстрока). Буст ограничен сверху, чтобы bm25 по-прежнему решал при слабом текстовом совпадении.
3. Итог обрезается до **`limit`**.

Пустой или только пробелы **`query`**: обработчик возвращает явное сообщение; `search_with_fallback` отдаёт пустой результат и предупреждение о пустом запросе.
