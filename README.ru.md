# TAUSIK

**Технический Агент Унифицированного Сопровождения, Инспекции и Контроля**

Фреймворк инженерного контроля для ИИ-агентов. Требует планирование перед кодом, доказательства перед закрытием задачи, сохраняет память между сессиями.

> Ваш ИИ-агент опять удалил боевой код? TAUSIK не позволит.

**Для кого:** разработчики и команды, использующие ИИ-агенты (Claude Code, Cursor, Windsurf) и желающие получать предсказуемый, проверяемый результат, а не надеяться на удачу.

## Проблема

ИИ-агенты для программирования мощные, но без принудительного контроля они:

- **Пропускают планирование** — сразу пишут код, не определив, что значит «готово»
- **Выдают желаемое за действительное** — заявляют, что задача выполнена, не запуская тесты и не проверяя критерии
- **Теряют контекст между сессиями** — каждый раз начинают с нуля, повторяя одни и те же ошибки
- **Повторяют неудачные подходы** — пробуют одно и то же сломанное решение трижды, потому что ничто не отслеживает, что уже не сработало

AGENTS.md и .cursorrules — это _рекомендации_, которые агент может проигнорировать. Нужен _принудительный контроль_.

## Решение

TAUSIK встраивается поверх вашей ИИ-среды разработки и обеспечивает инженерный процесс — не промптами, а жёсткими блокировками.

| Без TAUSIK | С TAUSIK |
|---|---|
| Агент сразу пишет код | Обязан определить цель и критерии приёмки до начала работы |
| Результат не проверяется | Закрытие задачи заблокировано, пока каждый критерий не подтверждён и все проверки не пройдены |
| Контекст теряется между сессиями | Проектная память сохраняет решения, закономерности и тупиковые подходы между сессиями |
| Одна и та же ошибка повторяется 3 раза | Неудачные подходы записываются — агент видит, что уже не сработало |
| «Тесты? Какие тесты?» | 15 проверок для вашего стека запускаются автоматически (pytest, ruff, tsc, eslint, cargo check, go vet...) |
| Нет видимости процесса | 6 метрик считаются автоматически: производительность, доля успеха с первой попытки, уровень дефектов и другие |

**Ключевое отличие:** quality gates TAUSIK — это _принудительный контроль_: CLI физически блокирует агента, не давая пропустить шаги.

## Быстрый старт

```bash
cd your-project
git submodule add https://github.com/Kibertum/tausik-core .tausik-lib
python .tausik-lib/bootstrap/bootstrap.py --smart --init my-project
```

`--smart` автоматически определяет ваш стек и включает подходящие проверки. `--init my-project` создаёт базу данных проекта (замените `my-project` на имя вашего проекта).

> **Важно:** После bootstrap перезапустите окно IDE (Claude Code, Cursor, Windsurf), чтобы MCP-серверы загрузились. Без перезапуска агент будет работать через CLI вместо MCP.

Скажите агенту в чате:

```
начинай работу
```

Агент откроет сессию, загрузит контекст проекта и покажет сводку. Дальше — просто опишите задачу:

```
исправь баг — кнопка не работает на мобильной версии
```

Агент создаст задачу, начнёт работу. Когда закончит — скажите `готово, отправляй`. Три сообщения — полный инженерный цикл с контролем качества.

## Как это работает

**Жизненный цикл задачи.** Каждая единица работы проходит через определённый цикл: `планирование -> старт -> реализация -> проверка -> завершение`. Агент не может пропускать этапы — quality gates контролируют последовательность.

**Quality gates.** Перед началом агент обязан определить, что он делает и как будет измеряться успех (QG-0: Context Gate). Перед закрытием — доказать, что каждый критерий выполнен: вывод тестов, шаги проверки (QG-2: Implementation Gate). Тесты и линтеры запускаются автоматически.

**Проектная память.** Решения, закономерности, соглашения и неудачные подходы хранятся в локальной SQLite-базе с полнотекстовым поиском. При старте новой сессии агент загружает этот контекст — больше никакого «кто это написал и зачем?» с нуля.

**Навыки (skills).** Навыки — это структурированные промпты, направляющие агента через сложные процессы. `/plan` декомпозирует задачу, `/review` запускает 5 параллельных агентов-ревьюеров, `/ship` делает ревью + тесты + коммит одним действием. Всего 33 навыка — от код-ревью до экспорта в Excel.

**MCP-инструменты.** 80 инструментов дают агенту программный доступ к базе проекта: создание задач, журналирование, поиск по памяти, управление сессиями. MCP (Model Context Protocol) — стандартный интерфейс между ИИ-агентами и внешними инструментами.

**Метрики.** TAUSIK отслеживает производительность, долю успеха с первой попытки, уровень пропущенных дефектов, время выполнения, долю тупиков и стоимость задачи — автоматически, без ручного ввода.

## Ключевые возможности

- **Любая ИИ-среда** — единый рабочий процесс в Claude Code, Cursor, Windsurf
- **Ноль зависимостей** — ядро на Python 3.11+ stdlib; MCP-зависимости устанавливаются автоматически в изолированный `.tausik/venv/`
- **Quality gates** — агент не начнёт без цели, не закроет без доказательств
- **Проектная память** — решения, закономерности и тупики сохраняются между сессиями (SQLite + FTS5)
- **15 проверок для вашего стека** — pytest, ruff, tsc, eslint, cargo check, go vet и другие запускаются автоматически
- **6 автоматических метрик** — производительность, FPSR, уровень дефектов, время выполнения — без ручного ввода
- **33 навыка** — `/plan`, `/ship`, `/review`, `/audit`, `/debug` и другие — структурированные сценарии для агента
- **80 MCP-инструментов** — полный программный доступ агента к базе проекта
- **Пакетное выполнение** — автономное выполнение многозадачных планов через `/run plan.md`

## Поддерживаемые среды разработки

| Среда | Инструменты | Навыки | Хуки (контроль в реальном времени) | Правила |
|-------|-------------|--------|-------------------------------------|---------|
| Claude Code | 80 инстр. | 33 навыка | контроль задач, фильтр команд, контроль push, автоформат | CLAUDE.md |
| Cursor | 80 инстр. | 33 навыка | — | .cursorrules |
| Windsurf | 80 инстр. | 33 навыка | — | .windsurfrules |
| Codex | — | — | — | только AGENTS.md |

**Хуки** — скрипты принудительного контроля в реальном времени: блокировка кода без задачи, опасных shell-команд, прямого push в main. Пока только Claude Code. В других средах те же MCP-инструменты и навыки — quality gates работают при `task start` и `task done`, но без предварительной блокировки действий. Codex имеет минимальную поддержку (только файл правил, без MCP).

## Dogfooding: TAUSIK создан с помощью себя

TAUSIK создавался с помощью самого себя с ранних стадий. Реальные цифры проекта:

| Метрика | Значение |
|---|---|
| Задач завершено | 266 |
| Сессий | 15 |
| Производительность | ~18 задач/сессию |
| Тестов | 918 |
| Зависимостей | 0 в ядре (MCP-deps в venv) |

Каждая новая возможность, каждая переработка, каждое исправление прошли через те же quality gates, которые поставляются с фреймворком.

## Методология

TAUSIK реализует [SENAR](https://senar.tech) ([GitHub](https://github.com/Kibertum/SENAR)) — открытый инженерный стандарт для разработки с помощью ИИ, определяющий quality gates, управление сессиями и метрики. Почему в TAUSIK именно такие правила? Потому что так определяет SENAR.
